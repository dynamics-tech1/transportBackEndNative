# Transport Management System work flow API Documentation

Note This document is used to the developers and testers to understand the flow of the application and how to use the API

## Documentation Structure

This documentation has been reorganized into focused Markdown files in the `docs/` directory for better maintainability:

- **[Setup & Initialization](docs/setup.md)** - Database setup and initial configuration
- **[Environment Configuration](docs/environment.md)** - Environment variables and settings  
- **[Authentication & Security](docs/auth.md)** - JWT tokens, RBAC, and security features
- **[Error Handling](docs/errors.md)** - API error responses and status codes
- **[User Management](docs/user-management.md)** - Admin and regular user operations
- **[Passenger Operations](docs/passenger-operations.md)** - Ride request creation and management
- **[Driver Registration](docs/driver-registration.md)** - Driver account creation and verification
- **[Document Management](docs/document-management.md)** - Document upload and approval processes
- **[Admin Operations](docs/admin-operations.md)** - Administrative functions and monitoring
- **[Finance Management](docs/finance-management.md)** - Deposits, subscriptions, and balance operations

## Getting Started

1. Review the [Setup Guide](docs/setup.md) for initial configuration
2. Configure your [Environment](docs/environment.md) variables
3. Check [Authentication](docs/auth.md) requirements for API access
4. Refer to specific sections as needed for API details and examples
